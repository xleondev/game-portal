# 2048 Game — Design Doc

**Date:** 2026-03-08
**Status:** Approved

## Overview

Add the classic 2048 sliding tile puzzle to the game portal. Standard 4×4 grid, no extra features, clean minimal visual style matching the original game.

## Approach

Vanilla HTML/CSS/JS — no framework or library dependencies. The original 2048 was built this way and it's the natural fit for a DOM-native grid puzzle.

## File Structure

```
public/games/2048/
  game/
    index.html   ← entry point (required by portal convention)
    style.css    ← board layout, tile colors, animations
    game.js      ← all game logic
  thumbnail.svg  ← SVG showing the board with warm beige palette
```

## Architecture

Board state is a flat 16-element array (4×4 row-major). 0 = empty cell. On each move:
1. Transform the state array (slide + merge)
2. Spawn a random tile (2 at 90%, 4 at 10%)
3. Re-render the DOM from state

### Core Functions

| Function | Responsibility |
|---|---|
| `move(direction)` | Slide and merge tiles in given direction |
| `addRandomTile()` | Place 2 or 4 on a random empty cell |
| `checkWin()` | Returns true if any cell equals 2048 |
| `checkGameOver()` | Returns true if no moves remain |
| `render()` | Rebuild grid DOM from current state |

### Input Handling

- **Desktop:** Arrow keys + WASD
- **Mobile:** Touch swipe (touchstart/touchend delta, min 30px threshold)

| Tile | Color |
|---|---|
| Empty | `#cdc1b4` |
| 2 | `#eee4da` |
| 4 | `#ede0c8` |
| 8 | `#f2b179` |
| 16 | `#f59563` |
| 32 | `#f67c5f` |
| 64 | `#f65e3b` |
| 128 | `#edcf72` |
| 256 | `#edcc61` |
| 512 | `#edc850` |
| 1024 | `#edc53f` |
| 2048 | `#edc22e` |

- Board background: `#bbada0`
- Page background: `#faf8ef`
- Tile pop-in via CSS `@keyframes` scale from 0 → 1
- Font: system-serif or "Clear Sans" (Google Fonts CDN)

### UI Elements

- Score counter at top
- "New Game" button
- Win overlay ("You reached 2048!") with "Keep Playing" and "New Game" buttons
- Game Over overlay with "New Game" button
